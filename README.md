# mimusic-plugin-musictag

MiMusic 音乐标签刮削插件，用于从在线音乐平台自动提取歌曲元数据。本插件基于 [MiMusic](https://github.com/mimusic-org/mimusic) 插件系统开发。

## 功能特性

- 🎵 **批量刮削**：支持按歌单批量刮削歌曲元数据
- 📝 **元数据提取**：自动获取歌曲标题、艺术家、专辑、歌词、封面图片
- 📊 **实时进度**：刮削过程中实时展示进度和状态
- 🔁 **失败重试**：支持对刮削失败的歌曲单独重试
- ⏹️ **随时停止**：支持中途停止刮削任务

## 技术栈

- **语言**：Go（编译目标为 WASI/WASM）
- **插件框架**：[mimusic-org/plugin](https://github.com/mimusic-org/plugin)
- **前端**：原生 HTML + CSS + JavaScript

## 项目结构

```
mimusic-plugin-musictag/
├── main.go                 # 插件入口，路由注册
├── handlers/
│   └── scraper.go          # HTTP 请求处理器
├── scraper/
│   ├── manager.go          # 刮削任务管理
│   ├── matcher.go          # 歌曲匹配逻辑
│   ├── types.go            # 数据类型定义
│   └── utils.go            # 工具函数
├── static/
│   ├── index.html          # 插件页面
│   ├── css/style.css       # 样式
│   └── js/app.js           # 前端逻辑
└── Makefile                # 构建脚本
```

## API 接口

| 方法 | 路径 | 描述 |
|------|------|------|
| `POST` | `/musictag/api/scrape/batch` | 启动批量刮削任务 |
| `GET` | `/musictag/api/scrape/status` | 查询刮削进度 |
| `POST` | `/musictag/api/scrape/stop` | 停止刮削任务 |
| `POST` | `/musictag/api/scrape/retry-failed` | 重试失败歌曲 |

## 构建

```bash
make build
```
